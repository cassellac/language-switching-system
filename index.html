JavaScript Controller - Part 1: Setup
üéôÔ∏è Narration
üó£Ô∏è Now we'll build the JavaScript controller that powers our language switching. We'll use a class-based approach to keep everything organized. The controller has three main responsibilities: initialize the system, handle language switching, and persist the user's choice. Let's start with the basic structure and constructor.
language-controller.js - Part 1
class LanguageController { // Constructor runs when class is instantiated constructor() { // Try to get saved language, default to English this.currentLang = localStorage.getItem('language') || 'en'; // Initialize the system when page loads this.init(); } // Initialize method sets up everything on page load init() { // Update all content to show correct language this.updateContent(this.currentLang); // Update button states to show active language this.updateUI(this.currentLang); // Update HTML lang attribute for accessibility this.updatePageLang(this.currentLang); } }
Understanding localStorage: localStorage is a browser feature that saves data even after the user closes the browser. We use it to remember which language the user selected, so they don't have to choose every time they visit.

JavaScript Controller - Part 3: Update Content
üéôÔ∏è Narration
üó£Ô∏è The update content method is where the actual text switching happens. This method finds every element on the page that has both data-en and data-es attributes. For each element, it reads the appropriate data attribute based on the selected language and updates the text content. The querySelector All method finds all matching elements at once, and we loop through them with forEach.
language-controller.js - Part 3
// Add this method to the LanguageController class updateContent(lang) { // Find all elements with both language attributes const elements = document.querySelectorAll('[data-en][data-es]'); // Loop through each element elements.forEach(el => { // Get the translation for selected language const content = el.getAttribute(`data-${lang}`); // If translation exists, update the element if (content) { // Check if it's an input or textarea if (el.tagName === 'INPUT' || el.tagName === 'TEXTAREA') { el.placeholder = content; } else { // For regular elements, update text content el.textContent = content; } } }); }

How querySelector Works:

[data-en][data-es] is a CSS selector that finds elements with BOTH attributes. The square brackets tell the browser to look for elements that have these specific data attributes.

Special Handling: Notice we check for input and textarea elements. These need their placeholder attribute updated instead of textContent.

JavaScript Controller - Part 4: Update UI
üéôÔ∏è Narration
üó£Ô∏è The update UI method handles the visual state of our language toggle buttons. When a user selects a language, we want that button to look active while the other looks inactive. We do this by adding and removing CSS classes. First, we remove the active class from all language buttons, then we add it to the currently selected button. This gives immediate visual feedback to the user.
language-controller.js - Part 4
// Add this method to the LanguageController class updateUI(lang) { // Find all language buttons (ids starting with 'lang-') const buttons = document.querySelectorAll('[id^="lang-"]'); // Remove active state from all buttons buttons.forEach(btn => { btn.classList.remove('active'); }); // Find and activate the selected language button const activeBtn = document.getElementById(`lang-${lang}`); if (activeBtn) { activeBtn.classList.add('active'); } }

JavaScript Controller - Part 5: Page Language
üéôÔ∏è Narration
üó£Ô∏è The update page lang method is small but important for accessibility. It updates the lang attribute on the HTML element. This tells browsers and screen readers what language the page content is in. Screen readers use this information to pronounce words correctly. Search engines also use it to understand your content better. This is a simple one-line method but it makes a big difference for accessibility.
language-controller.js - Part 5
// Add this method to the LanguageController class updatePageLang(lang) { // Update the HTML element's lang attribute document.documentElement.lang = lang; } // Optional but recommended: Announce changes to screen readers announceChange(lang) { const announcement = lang === 'es' ? 'Idioma cambiado a espa√±ol' : 'Language changed to English'; // Create temporary announcement element const announcer = document.createElement('div'); announcer.setAttribute('aria-live', 'polite'); announcer.setAttribute('aria-atomic', 'true'); announcer.className = 'sr-only'; announcer.textContent = announcement; // Add to page, then remove after screen reader announces document.body.appendChild(announcer); setTimeout(() => document.body.removeChild(announcer), 1000); }
ARIA Accessibility: The aria-live attribute tells screen readers to announce content changes. Setting it to "polite" means it waits for the user to finish what they're doing before announcing.

Initialize the System
üéôÔ∏è Narration
üó£Ô∏è Now we need to actually create an instance of our Language Controller class and make the toggle language function available globally so our HTML buttons can call it. We wait for the DOM content loaded event to ensure all HTML elements are ready before we initialize the controller. Then we create a global function that calls the controller's toggle language method.
language-controller.js - Final Part
// Wait for page to fully load let languageController; document.addEventListener('DOMContentLoaded', () => { // Create instance of the controller languageController = new LanguageController(); }); // Global function for HTML onclick handlers function toggleLanguage(lang) { if (languageController) { languageController.toggleLanguage(lang); } }

Why Use DOMContentLoaded?

This event fires when the HTML is fully loaded and parsed. If we try to access HTML elements before this, they might not exist yet and our code will break.

Performance Optimization Tips
üéôÔ∏è Narration
üó£Ô∏è While our system is already efficient, here are some optimization tips for larger sites. Consider lazy loading translations for pages with lots of content. Cache the DOM queries so you're not searching the entire document every time. Use event delegation for language buttons instead of individual onclick handlers. Minify your JavaScript file for production. These optimizations become more important as your site grows.
Performance Optimizations
class LanguageController { constructor() { this.currentLang = localStorage.getItem('language') || 'en'; // Cache DOM queries this.translatableElements = null; this.languageButtons = null; this.init(); } cacheElements() { // Store element references to avoid repeated queries if (!this.translatableElements) { this.translatableElements = document.querySelectorAll('[data-en][data-es]'); } if (!this.languageButtons) { this.languageButtons = document.querySelectorAll('[id^="lang-"]'); } } updateContent(lang) { this.cacheElements(); // Use cached elements this.translatableElements.forEach(el => { const content = el.getAttribute(`data-${lang}`); if (content) el.textContent = content; }); } }
Performance Tip: Caching DOM queries can improve performance by 50% or more on pages with many translatable elements.

Troubleshooting Common Issues
üéôÔ∏è Narration
üó£Ô∏è Let's cover some common issues you might encounter. If text doesn't update, check that you have both data attributes on every element. If buttons don't work, verify your JavaScript is loaded and the global toggle language function exists. If language doesn't persist, check that localStorage is enabled in the browser. If some elements don't switch, make sure you're querying for the right attributes. These are the most frequent problems and their solutions.
Problem Cause Solution
Text doesn't switch Missing data attributes Add both data-en and data-es to element
Buttons do nothing JavaScript not loaded Check script tag and console for errors
Language doesn't save localStorage disabled Check browser settings or use cookies
Some text stays English Wrong querySelector Verify elements have correct attributes
Page refreshes Button is inside a form Add type="button" to prevent submission
