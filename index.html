3
HTML Foundation with Data Attributes
üéôÔ∏è Narration
The foundation of our system is HTML data attributes. Each element needing translation will have two custom data attributes: data-en for English and data-es for Spanish. When a user clicks a language button, JavaScript reads these attributes and updates the visible text. This approach works with any HTML element.
index.html - Basic Structure
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title data-en="Digital Allies - Web Development" data-es="Digital Allies - Desarrollo Web"> Digital Allies - Web Development </title> <link rel="stylesheet" href="styles.css"> </head> <body> <!-- Language Toggle Buttons --> <div class="language-switcher"> <button onclick="toggleLanguage('en')" id="lang-en" class="lang-btn active"> EN </button> <button onclick="toggleLanguage('es')" id="lang-es" class="lang-btn"> ES </button> </div> <script src="language-controller.js"></script> </body> </html>

Understanding Data Attributes:

Data attributes are custom HTML attributes that start with "data-". They let you store extra information on HTML elements without affecting how the page looks. JavaScript can read these attributes to get the translated text.

5
JavaScript Controller - Class Structure
üéôÔ∏è Narration
Now we build the brain of our system: the Language Controller class. The constructor checks localStorage to see if the user has a saved language preference. If they do, we use that. If not, we default to English. Then we call the init method to set up the page with the correct language.
language-controller.js - Part 1
class LanguageController { constructor() { // Try to get saved language from browser storage // If nothing saved, default to 'en' this.currentLang = localStorage.getItem('language') || 'en'; // Initialize the system when page loads this.init(); } init() { // Update all content to show correct language this.updateContent(this.currentLang); // Update button states this.updateUI(this.currentLang); // Update HTML lang attribute this.updatePageLang(this.currentLang); } }

Why Use localStorage?

localStorage is a browser feature that saves data permanently. When a user selects Spanish, we save that choice. The next time they visit your website (even days later), they'll see Spanish automatically.

6
The Toggle Language Function
üéôÔ∏è Narration
The toggleLanguage method runs when a user clicks EN or ES. It does four things: updates the currentLang variable, saves that choice to localStorage, updates all content on the page, and updates button states. All of this happens instantly without reloading the page.
language-controller.js - Part 2
toggleLanguage(lang) { // Update the current language variable this.currentLang = lang; // Save choice to browser storage localStorage.setItem('language', lang); // Update all content on the page this.updateContent(lang); // Update button appearance this.updateUI(lang); // Update HTML lang attribute this.updatePageLang(lang); }

Update Content - Translation Engine
üéôÔ∏è Narration
The updateContent method is where the magic happens. This function actually switches the text on your page. It uses querySelectorAll to find every element with both data-en and data-es attributes. Then it loops through each element, reads the data attribute for the selected language, and updates the element's text content.
language-controller.js - Part 3
updateContent(lang) { // Find ALL elements with both language attributes const elements = document.querySelectorAll('[data-en][data-es]'); // Loop through each element elements.forEach(el => { // Get translation for selected language const content = el.getAttribute(`data-${lang}`); if (content) { // Special handling for form fields if (el.tagName === 'INPUT' || el.tagName === 'TEXTAREA') { el.placeholder = content; } else { // For all other elements el.textContent = content; } } }); }
Understanding querySelectorAll: The selector '[data-en][data-es]' finds elements with BOTH attributes. This is standard CSS selector syntax.
8
Update UI - Button States
üéôÔ∏è Narration
The updateUI method handles the visual state of language buttons. When a user selects a language, we want that button to look active. We remove the active class from all buttons, then add it to the currently selected button. This gives immediate visual feedback.
language-controller.js - Part 4
updateUI(lang) { // Find all language buttons const buttons = document.querySelectorAll('[id^="lang-"]'); // Remove active state from ALL buttons buttons.forEach(btn => { btn.classList.remove('active'); }); // Find button for selected language const activeBtn = document.getElementById(`lang-${lang}`); // Add active class if (activeBtn) { activeBtn.classList.add('active'); } }

CSS Classes for Visual Feedback:

The 'active' class is defined in CSS. It changes the button's appearance (background color, shadow) to show which language is currently selected.

Accessibility Features
üéôÔ∏è Narration
Accessibility is critical. The updatePageLang method updates the lang attribute on the HTML element, which tells screen readers what language the page is in. This helps screen readers pronounce words correctly and creates a better experience for users with assistive technology.
language-controller.js - Part 5
updatePageLang(lang) { // Update HTML lang attribute document.documentElement.lang = lang; } // Optional: Announce changes to screen readers announceChange(lang) { const announcement = lang === 'es' ? 'Idioma cambiado a espa√±ol' : 'Language changed to English'; // Create temporary announcement element const announcer = document.createElement('div'); announcer.setAttribute('aria-live', 'polite'); announcer.setAttribute('aria-atomic', 'true'); announcer.className = 'sr-only'; announcer.textContent = announcement; document.body.appendChild(announcer); setTimeout(() => document.body.removeChild(announcer), 1000); }
ARIA Live Regions: ARIA provides attributes that make dynamic content changes accessible to screen reader users. The sr-only class hides the element visually but keeps it available to screen readers.
10
Initialize the Controller
üéôÔ∏è Narration
Now we create an instance of our LanguageController class and make the toggleLanguage function globally available. We listen for the DOMContentLoaded event, which fires when all HTML elements are ready. Then we create the controller instance and set up the global function for HTML onclick handlers.
language-controller.js - Final Part
let languageController; // Wait for page to fully load document.addEventListener('DOMContentLoaded', () => { // Create controller instance languageController = new LanguageController(); }); // Global function for HTML onclick handlers function toggleLanguage(lang) { if (languageController) { languageController.toggleLanguage(lang); } }

Why DOMContentLoaded?

If we try to access HTML elements before the page loads, they won't exist yet and our code will fail. DOMContentLoaded ensures everything is ready first.

Performance Tips
üéôÔ∏è Narration
While our system is already fast, here are optimization tips for larger sites. Cache DOM queries so you're not searching the entire document every time. Store references to frequently accessed elements in variables. Minify your JavaScript file for production to reduce file size.
Performance-Optimized Version
class LanguageController { constructor() { this.currentLang = localStorage.getItem('language') || 'en'; // Cache DOM queries this.translatableElements = null; this.languageButtons = null; this.init(); } cacheElements() { if (!this.translatableElements) { this.translatableElements = document.querySelectorAll('[data-en][data-es]'); } if (!this.languageButtons) { this.languageButtons = document.querySelectorAll('[id^="lang-"]'); } } updateContent(lang) { this.cacheElements(); this.translatableElements.forEach(el => { const content = el.getAttribute(`data-${lang}`); if (content) { if (el.tagName === 'INPUT' || el.tagName === 'TEXTAREA') { el.placeholder = content; } else { el.textContent = content; } } }); } }
Performance Impact: Caching DOM queries can improve performance by 50% or more on pages with many translatable elements.

Common Issues & Solutions
üéôÔ∏è Narration
Let's cover common issues you might encounter. If text doesn't update, check that every element has both data-en and data-es attributes with no typos. If buttons don't work, verify your JavaScript file is loaded by checking the Network tab in DevTools. If language doesn't persist, check that localStorage is enabled in browser settings.

Troubleshooting Guide:

Problem Cause Solution
